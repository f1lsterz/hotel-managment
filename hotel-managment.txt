Ось детальний перелік функціональних вимог для веб-застосунку для підтримки роботи готелю, а також опис ролей користувачів:
1. Функціональні вимоги
1.1. Реєстрація та автентифікація
•	Реєстрація користувачів:
o	Реєстрація адміністратора, персоналу, гостей.
o	Для адміністратора та персоналу – верифікація через email, пароль.
o	Гість може реєструватися через просту форму для бронювання або логін через соціальні мережі.
•	Логін/Вихід:
o	Авторизація через email і пароль або через соціальні мережі (Google, Facebook).
o	Генерація JWT токенів для забезпечення сесій.
•	Відновлення пароля:
o	Відновлення пароля через email з одноразовим посиланням.
1.2. Управління користувачами
•	Адміністратор:
o	Створення, редагування та видалення користувачів (персонал, гості).
o	Призначення ролей (адміністратор, менеджер, ресепшн).
o	Перегляд активності користувачів, включаючи історію бронювань.
•	Менеджер/Персонал:
o	Перегляд своїх особистих даних, можливість їх редагування.
o	Зміна статусу бронювання, обробка запитів.
1.3. Управління номерами
•	Створення та редагування номерів:
o	Додавання, редагування та видалення номерів.
o	Визначення типу номеру (одномісний, двомісний, люкс тощо).
o	Налаштування цін та доступності номерів.
•	Перегляд доступних номерів:
o	Перегляд списку номерів з можливістю фільтрації за ціною, типом і статусом (вільний, заброньований, на ремонті).
•	Налаштування особливостей номерів:
o	Визначення послуг для кожного номера (безкоштовний Wi-Fi, кондиціонер, сніданок тощо).
1.4. Бронювання
•	Форма бронювання:
o	Гості можуть вибирати дату заїзду, дату виїзду, тип номеру.
o	Вибір додаткових послуг, таких як трансфер, сніданок, екскурсії тощо.
o	Розрахунок загальної вартості.
•	Бронювання номера:
o	Гість може здійснити бронювання через веб-застосунок, сплатити онлайн або підтвердити бронювання пізніше.
o	Підтвердження бронювання через email.
•	Перегляд історії бронювань:
o	Користувачі можуть переглядати історію своїх бронювань, з можливістю анулювання чи редагування.
1.5. Розрахунки та платежі
•	Інтеграція з платіжними системами:
o	Реалізація онлайн-платежів через системи, такі як Stripe або PayPal.
o	Оформлення та обробка платежів за бронювання.
•	Перегляд платіжної історії:
o	Гість може переглядати свої транзакції, рахунки та історію оплат.
o	Адміністратор та менеджер можуть переглядати платіжні дані для кожного користувача.
1.6. Відгуки
•	Створення відгуків:
o	Гості можуть залишати відгуки про номери після перебування.
o	Вказівка рейтингу (1–5) і текстового коментаря.
•	Перегляд відгуків:
o	Відгуки доступні для гостей (у профілі номеру) та адміністрації.
o	Сортування за рейтингом, датою.
•	Адміністрування відгуків:
o	Адміністратор може видаляти відгуки з неналежним змістом.
1.7. Знижки
•	Управління знижками:
o	Адміністратор може створювати знижки на певні номери.
o	Визначення періоду дії, відсотка знижки та опису.
•	Автоматичне застосування знижок:
o	Розрахунок вартості номерів з урахуванням активних знижок.
•	Перегляд доступних знижок:
o	Гості можуть бачити активні знижки у списку номерів.
•	Історія знижок:
o	Адміністратор може переглядати всі створені знижки та їх ефективність.
1.8. Повідомлення та сповіщення
•	Для гостей:
o	Сповіщення про бронювання, зміни статусу, доступність номерів і знижки.
•	Для персоналу:
o	Повідомлення про нові бронювання, платежі, статуси номерів.
•	Для адміністратора:
o	Сповіщення про ключові події (генерація звітів, нові знижки тощо).
1.9. Звіти та аналітика
•	Адміністративні звіти:
o	Генерація звітів по доходах, зайнятості номерів, ефективності знижок.
•	Статистика відгуків:
o	Аналіз середнього рейтингу номерів.
•	Оцінка бронювань:
o	Перегляд кількості бронювань по типах номерів, датах тощо.
1.10. Чат з адміністрацією (опціонально)
•	Гість:
o	Звернення до персоналу через вбудований чат.
•	Адміністратор і персонал:
o	Обробка запитів через чат.

2. Ролі користувачів
2.1. Зареєстрований користувач
•	Основні можливості:
o	Реєстрація та вхід до системи.
o	Перегляд доступних номерів, бронювання номерів.
o	Оформлення платежів за бронювання.
o	Перегляд історії бронювань.
o	Отримання сповіщень про бронювання та зміни статусу.
2.2. Персонал (Ресепшн)
•	Основні можливості:
o	Перегляд доступних номерів і управління їхнім статусом.
o	Управління бронюваннями (створення, редагування, скасування).
o	Перегляд та управління інформацією про гостей.
o	Спілкування з гостями (якщо реалізовано через чат).
2.4. Адміністратор
•	Основні можливості:
o	Повний доступ до всіх даних та налаштувань.
o	Управління ролями користувачів (створення, редагування, видалення).
o	Генерація фінансових звітів.
o	Перегляд та аналіз історії бронювань, платежів, статистики.
o	Управління номерним фондом та іншими налаштуваннями готелю.
Цей перелік функціональних вимог дозволяє створити структуру додатку для готельного бізнесу з чітким розподілом ролей та прав доступу.
